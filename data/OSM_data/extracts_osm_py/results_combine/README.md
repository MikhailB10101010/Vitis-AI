# Функция объединения GeoJSON файлов

## Назначение

Скрипт `many_geojson_to_one.py` предназначен для рекурсивного поиска, загрузки и объединения всех `.geojson` файлов из указанной папки и всех её подпапок в один общий GeoJSON файл.

## Основные действия

- **Рекурсивный обход** — поиск файлов во всех вложенных директориях.
- **Фильтрация колонок** — возможность оставить только нужные атрибуты (например, `osm_id`, `name`, `geometry`).
- **Отслеживание источника** — добавление колонки `source_file` с именем исходного файла.
- **Очистка данных**:
  - Удаление дубликатов по полю `osm_id`.
  - Удаление записей с пустой геометрией.
  - Удаление node ("osm_type": "node"), way остаются.
  - Валидация и исправление геометрии (метод `make_valid()`).
- **Статистика процесса** — вывод количества найденных/обработанных файлов, объектов и колонок.
- **Обработка ошибок** — продолжение работы при проблемах с отдельными файлами с подсчётом количества ошибок.

## Используемые библиотеки

- `geopandas` — работа с геоданными.
- `pandas` — объединение таблиц.
- `pathlib` — удобная работа с путями файловой системы.

## Параметры функции

| Параметр | Тип | Описание |
|----------|-----|----------|
| `root_folder` | `str` или `Path` | Путь к корневой папке для поиска `.geojson` файлов |
| `output_file` | `str` | Имя выходного файла (по умолчанию `'merged.geojson'`) |
| `select_cols` | `list` | Список колонок, которые нужно оставить (обязательно включает `'geometry'`) |

## Выходные данные

- Создаётся файл `merged.geojson` (или указанное имя) в текущей рабочей директории.
- В консоль выводится:
  - Количество найденных `.geojson` файлов.
  - Статус загрузки каждого файла (имя, количество объектов и колонок).
  - Итоговая статистика (общее количество объектов, число колонок).
  - Количество ошибок при чтении (если были).
  - Размер сохранённого файла в мегабайтах.

## Примечания

- Функция автоматически добавляет колонку `source_file`, если её нет в исходных данных.
- Если не указан `select_cols`, сохраняются **все** колонки из исходных файлов.
- Геометрия приводится к валидному состоянию через `make_valid()`.
- Дубликаты удаляются **только** по полю `osm_id` - убедитесь, что этот идентификатор присутствует и уникален для каждого объекта.
- При указании `select_cols` колонка `geometry` добавляется автоматически, даже если не включена в список.
